<p dir="auto">This is the first part of the AirBnB clone project where we worked on the backend of the project whiles interfacing it with a console application with the help of the cmd module in python.</p>

<h2 dir="auto"><a id="user-content-description-of-the-command-interpreter" class="anchor" aria-hidden="true" href="#description-of-the-command-interpreter"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Description of the command interpreter:</h2>
<p dir="auto">The interface of the application is just like the Bash shell except that this has a limited number of accepted commands that were solely defined for the purposes of the usage of the AirBnB website.</p>
<p dir="auto">This command line interpreter  serves as the frontend of the web app where users can interact with the backend which was developed with python OOP programming.</p>
<p dir="auto">Some of the commands available are:</p>
<ul dir="auto">
<li>show</li>
<li>create</li>
<li>update</li>
<li>destroy</li>
<li>count</li>
</ul>
<p dir="auto">And as part of the implementation of the command line interpreter coupled with the backend and file storage system, the folowing actions can be performed:</p>
<ul dir="auto">
<li>Creating new objects (ex: a new User or a new Place)</li>
<li>Retrieving an object from a file, a database etc…</li>
<li>Doing operations on objects (count, compute stats, etc…)</li>
<li>Updating attributes of an object</li>
<li>Destroying an object</li>
</ul>

<p dir="auto">After cloning the repository you will have a folder called AirBnB_clone. In here there will be several files that allow the program to work.</p>
<blockquote>
<p dir="auto">/console.py : The main executable of the project, the command interpreter.</p>
<p dir="auto">models/engine/file_storage.py: Class that serializes instances to a JSON file and deserializes JSON file to instances</p>
<p dir="auto">models/__ init __.py:  A unique <code>FileStorage</code> instance for the application</p>
<p dir="auto">models/base_model.py: Class that defines all common attributes/methods for other classes.</p>
<p dir="auto">models/user.py: User class that inherits from BaseModel</p>
<p dir="auto">models/state.py: State class that inherits from BaseModel</p>
<p dir="auto">models/city.py: City class that inherits from BaseModel</p>
<p dir="auto">models/amenity.py: Amenity class that inherits from BaseModel</p>
<p dir="auto">models/place.py: Place class that inherits from BaseModel</p>
<p dir="auto">models/review.py: Review class that inherits from BaseModel</p>
</blockquote>

<h2 dir="auto"><a id="user-content-format-of-command-input" class="anchor" aria-hidden="true" href="#format-of-command-input"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Format of Command Input</h2>
<p dir="auto">In order to give commands to the console, these will need to be piped through an echo in case of  <strong>Non-interactive mode</strong>.</p>
<p dir="auto">In  <strong>Interactive Mode</strong>  the commands will need to be written with a keyboard when the prompt appears and will be recognized when an enter key is pressed (new line). As soon as this happens, the console will attempt to execute the command through several means or will show an error message if the command didn't run successfully. In this mode, the console can be exited using the <strong>CTRL + D</strong> combination,  <strong>CTRL + C</strong>, or the command <strong>quit</strong> or <strong>EOF</strong>.</p>
